Pine+ctrl ×
Pine+ctrl is a centralised IoT control system. It can:
  • Store collated and combined status logs from multiple devices.
  • Act as a file upload/download system for centralised cloud file storage.
  • Act as a file fetching proxy.
  • Act as a gateway to the internet from an intranet.

What is Pine+ctrl?

Pine+ctrl is a centralised IoT control system. It is another helping hand that you might need for your IoT device.

More Operations ×
Example:
  • GET Request ?m=check&v=info.txt

  • Possible outputs
    • SUCCESS: ./raya/info.txt has been deleted!

    • ERROR: ./raya/info.txt cannot be found
  • All files are stored in the ./raya/ directory. File uploads must be through multipart POST form data.
Misc. operations:
  • Echo text m= echo v= [text]

  • Echo IP Address m= ip

Why use Pine+ctrl?

Well, because

  • It is a centralized IoT control system, all connected devices can share files, combined storage, basically it is a simple cloud system.
  • It allows debugging of large computing nodes networks such as renderfarms, all nodes store logs in the same place on the server so can be collectively viewed
  • It has a friendly UI for admins, it is lightweight and has easy integration with multiple systems.

Operations

Requests:

  • Upload file m= upload v= [file_name]

  • Check file m= check v= [file_name]

  • Download file m= down v= [file_name]

  • Delete file m= delete v= [file_name]

Setting up ×
Authentication:
  • Your requests must contain the following parameters:
    • id=[device_id]

    • key=[auth_key]
Example:
  • index.php?id=node_1&key=randompass1

Setting up

Config:
  • Edit values at the top of index.php
Users:
  • Edit the array in auth.php and add an id and key for each device or user which will be accessing Pine+ctrl.
